# VATOS PPT Design Guide

## Purpose

이 문서는 VATOS 공통 디자인 기준을 기반으로, PPT 산출물에서 자주 발생하는 레이아웃 치우침, 콘텐츠 상단 쏠림, 과도한 하단 여백, 텍스트 넘침, 표·카드 깨짐 및 슬라이드 밀도 문제를 방지하기 위한 문서 유형별 세부 규격이다.

공통 브랜드 컬러, 로고, 타이포그래피, 디자인 톤, Do/Don't 및 수정 가능성 원칙은 저장소의 공통 디자인 기준을 따른다.

본 문서는 공통 디자인 원칙을 반복하기보다, PPT 제작 과정에서 발생하기 쉬운 화면 구성과 출력 안정성 문제를 예방하는 데 목적이 있다.

PPT 작업 시 본 가이드의 규칙을 우선 적용하고, 실제 요소의 위치, 간격, 크기, 정렬감 및 레이아웃 구성은 저장소의 PPT 참고자료를 함께 확인한다.

참고자료는 내용 작성용 원본이 아니라 레이아웃과 시각적 균형을 확인하기 위한 자료로 사용한다.

표지, 섹션 구분, Closing 및 테마 적용이 필요한 장표는 `theme-usage-rules` 역할의 가이드를 함께 적용한다. 승인된 테마 예시는 고정 템플릿이 아니라 컬러 균형, 깊이감, 여백, 타이포그래피 위계와 전문성을 판단하는 품질 기준으로 사용한다.

## 0. PPT Theme Application Rules

VATOS PPT 테마는 특정 표지 레이아웃 하나를 반복해서 사용하는 고정 템플릿이 아니다.

브랜드 컬러 역할과 시각적 품질은 유지하되, 문서 목적, 대상 독자, 제목 길이 및 정보량에 따라 표지와 섹션 장표의 구도와 조형 방식을 다양하게 구성한다.

### 0.1 Theme Visual Language

- Urban Navy는 표지, 섹션 구분, 핵심 메시지 및 Closing에 사용할 수 있다.
- 정보량이 많은 본문 슬라이드는 Light Theme를 우선 적용한다.
- Premium Dark는 메인 배경이 아니라 깊이감, 면 분할 또는 보조 레이어에 제한적으로 사용한다.
- Vatos Cyan은 얇은 선, 짧은 구분선, 작은 라벨 또는 핵심 포인트로 제한한다.
- Urban은 도시 사진, 건축물, 도로, 야경 또는 스카이라인을 의미하지 않는다.
- Urban은 정돈된 비대칭 구성, 깊은 네이비 명암, 구조적인 기하 표현, 현대적인 여백과 타이포그래피를 의미한다.

### 0.2 Cover Motif Rules

표지에는 아래 조형 방식 중 문서 목적에 적합한 한 가지를 중심으로 사용할 수 있다.

- Diagonal Layer
- Vertical Band
- Offset Frame
- Tonal Block
- Thin Line Grid
- Typography Focus
- Cropped Geometry

적용 시 아래 규칙을 따른다.

- 한 표지에는 주요 조형 방식 하나만 선택한다.
- 사선 레이어는 선택 가능한 방식 중 하나이며 필수 요소가 아니다.
- 승인 예시의 제목 위치, 도형 비율, 사선 위치 및 면 분할을 그대로 반복 복제하지 않는다.
- Cover, Section Divider, Closing은 같은 시각 언어를 공유하되 동일한 구도를 반복하지 않는다.
- 네이비 직사각형 하나만 배치하여 표지를 완성하지 않는다.
- Dark Theme와 Light Theme를 단순한 색상 견본 카드처럼 표현하지 않는다.
- 표지의 핵심 시각 요소를 일반 카드 컴포넌트로 대체하지 않는다.

### 0.3 Internal Slide Theme Rules

- 본문은 밝은 배경, 네이비 제목, 얇은 보더와 제한적인 Cyan 포인트를 기본으로 한다.
- 본문 슬라이드를 일률적인 카드 나열형으로 구성하지 않는다.
- 카드는 병렬 요약, 핵심 수치, 기능 또는 분류 정보처럼 독립된 항목을 비교할 때 사용한다.
- 카드가 적합하지 않은 정보는 표, 차트, 비교 구조, 프로세스, 다이어그램 또는 직접 배치형 레이아웃으로 구성한다.
- 같은 문서 안에서도 모든 본문 슬라이드를 동일한 3개 또는 4개 카드 구조로 반복하지 않는다.
- 구성비, 추이, 관계, 상세 데이터는 카드보다 차트, 표 또는 다이어그램을 우선 검토한다.

## 1. Slide Layout Stability Rules

PPT 슬라이드는 제목 영역과 본문 영역을 구분하여 구성한다.

모든 주요 콘텐츠는 슬라이드 중앙의 안전 영역 안에 배치해야 하며, 의도 없는 좌우 치우침이나 제목 아래로 콘텐츠가 몰리는 구성을 만들지 않는다.

### 1.1 Slide Canvas Rule

- 기본 슬라이드 비율은 16:9를 기준으로 한다.
- 주요 콘텐츠는 슬라이드의 가로 중앙축과 본문 영역의 세로 중심을 함께 고려하여 배치한다.
- 표지, 섹션 구분, Closing을 제외한 일반 본문 슬라이드는 중앙 안전 영역을 사용한다.
- 전체 콘텐츠 블록이 화면 한쪽으로 치우쳐 보이지 않도록 한다.
- 좌측 정렬 텍스트를 사용하더라도 콘텐츠 그룹 전체는 슬라이드 중앙 안전 영역 안에 배치한다.
- 장식 요소, 사선 레이어, 배경 패턴은 화면 바깥까지 확장할 수 있으나 본문 텍스트와 핵심 정보는 안전 영역 안에 유지한다.
- 배경 장식, 로고, 페이지 번호 또는 섹션 라벨을 핵심 콘텐츠의 정렬 기준으로 사용하지 않는다.

### 1.2 Slide Area Separation

일반 본문 슬라이드는 아래 영역을 시각적으로 구분하여 구성한다.

1. 제목 영역
2. 본문 콘텐츠 영역
3. 하단 보조 정보 영역

- 제목 영역에는 슬라이드 제목, 섹션 라벨 및 필요한 경우 짧은 부제만 배치한다.
- 본문 콘텐츠 영역에는 카드, 표, 차트, 다이어그램, 목록 등 해당 슬라이드의 핵심 콘텐츠를 배치한다.
- 하단 보조 정보 영역에는 단위, 출처, 주석, 페이지 번호 등 보조 정보만 배치한다.
- 제목 바로 아래에서 본문 콘텐츠를 순차적으로 쌓는 문서형 배치를 기본값으로 사용하지 않는다.
- 제목을 제외한 나머지 사용 가능 영역 전체를 본문 콘텐츠 영역으로 판단한다.
- 본문 콘텐츠는 제목 아래의 첫 번째 빈 공간부터 채우는 것이 아니라, 본문 영역 전체의 균형을 기준으로 위치를 결정한다.

### 1.3 Safe Area

- 상단, 하단, 좌우 가장자리에는 충분한 여백을 둔다.
- 핵심 텍스트, 표, 카드 및 차트는 슬라이드 가장자리에 붙이지 않는다.
- 일반 본문 슬라이드의 콘텐츠 그룹은 슬라이드 너비의 약 80~86% 안에서 구성한다.
- 표와 카드 그리드는 슬라이드 중앙을 기준으로 균형 있게 배치한다.
- 본문 콘텐츠의 좌우 여백은 시각적으로 비슷하게 유지한다.
- 제목 영역과 본문 콘텐츠 영역 사이에는 명확한 간격을 둔다.
- 본문 콘텐츠와 슬라이드 하단 사이에도 충분한 여백을 확보한다.
- 표지, 섹션 구분 및 Closing 장표는 디자인 목적에 따라 더 넓은 배경 영역을 사용할 수 있으나 핵심 텍스트는 안전 영역 안에 둔다.

### 1.4 Horizontal Alignment Rules

- 제목은 좌측 정렬할 수 있지만 제목 블록 자체는 슬라이드 안전 영역 안에 배치한다.
- 카드, 표, 차트는 동일한 좌우 여백을 기준으로 정렬한다.
- 2열 레이아웃은 좌우 컬럼의 시각적 무게가 균형을 이루도록 구성한다.
- 두 컬럼은 동일한 상단 기준선을 사용한다.
- 한쪽 컬럼에만 과도하게 많은 콘텐츠를 배치하지 않는다.
- 3열 또는 4열 카드 레이아웃은 개별 카드가 아니라 카드 그룹 전체를 중앙 정렬한다.
- 표는 슬라이드 중앙에 배치하고 좌우 여백이 비슷하게 보이도록 한다.
- 차트는 전체 플롯 영역과 범례를 포함한 시각적 경계를 기준으로 중앙 배치한다.
- 로고, 페이지 번호 및 섹션 라벨은 보조 요소이며 핵심 콘텐츠의 정렬 기준을 흐트러뜨리지 않는다.

### 1.5 Vertical Balance Rules

PPT 본문 슬라이드는 좌우 정렬뿐 아니라 상하 균형을 반드시 함께 고려한다.

콘텐츠 양이 적은 슬라이드는 핵심 콘텐츠를 제목 바로 아래에 붙이지 않고, 제목을 제외한 본문 영역의 세로 중앙에 배치한다.

- 일반 본문 슬라이드는 제목 영역과 본문 콘텐츠 영역을 명확히 분리한다.
- 제목 아래 남은 영역 전체를 사용 가능한 본문 콘텐츠 영역으로 판단한다.
- 콘텐츠 양이 적을 경우 카드·표·차트·다이어그램 그룹을 본문 영역 상단에 붙이지 않는다.
- 콘텐츠 그룹의 상단 여백과 하단 여백이 시각적으로 유사하도록 배치한다.
- 본문 콘텐츠 그룹은 슬라이드 전체가 아니라 제목 영역을 제외한 본문 영역을 기준으로 세로 중앙 정렬한다.
- 필요하면 본문 영역의 정확한 중앙보다 약간 위에 배치할 수 있으나, 제목 바로 아래에 붙어 보이면 안 된다.
- 슬라이드 하단의 빈 공간이 본문 콘텐츠 위쪽 여백보다 과도하게 넓어 보이지 않도록 한다.
- 제목과 설명문은 상단에 배치할 수 있으나, 핵심 콘텐츠 그룹까지 제목 아래에서 연속적으로 쌓지 않는다.
- 하단에 주석, 단위 또는 페이지 번호만 있는 경우에도 본문 콘텐츠 그룹은 하단 공간을 고려하여 균형 있게 배치한다.
- 내용이 적은 요약 슬라이드는 콘텐츠 그룹을 본문 영역의 세로 중앙 또는 중앙보다 약간 위에 배치한다.
- 내용이 많은 상세 슬라이드는 상단 정렬을 사용할 수 있으나, 콘텐츠가 본문 영역 하단까지 자연스럽게 이어져야 한다.
- 콘텐츠가 본문 영역 높이의 절반 이하인 경우에는 기본적으로 세로 중앙 배치를 우선 검토한다.
- 콘텐츠가 본문 영역 대부분을 사용하는 경우에만 상단 정렬을 허용한다.

#### Do Not

- 제목 바로 아래에서부터 모든 콘텐츠를 순서대로 쌓지 않는다.
- 콘텐츠 양이 적은데도 상단 정렬을 사용하여 슬라이드 하단을 과도하게 비우지 않는다.
- 카드 3개 또는 4개만 있는 슬라이드를 제목 아래에 붙여 배치하지 않는다.
- 표나 차트의 실제 높이가 낮다는 이유로 본문 영역 상단에 배치하지 않는다.
- 페이지 번호나 하단 로고가 있다는 이유로 하단 여백을 사용한 것으로 판단하지 않는다.
- 각 객체를 개별적으로 중앙 정렬하면서 콘텐츠 그룹 전체는 위쪽에 치우치는 구성을 만들지 않는다.

### 1.6 Content Group Positioning Rule

콘텐츠 배치는 개별 객체가 아니라 관련 요소를 묶은 콘텐츠 그룹 단위로 판단한다.

- 제목, 부제, 본문, 카드, 주석을 각각 따로 중앙 정렬하지 않는다.
- 같은 의미를 구성하는 카드, 수치, 아이콘, 설명문은 하나의 그룹으로 간주한다.
- 콘텐츠 그룹 전체의 외곽 경계를 기준으로 가로·세로 위치를 판단한다.
- 카드 내부 텍스트의 정렬과 카드 그룹 자체의 슬라이드 배치는 별도로 판단한다.
- 개별 객체가 중앙에 있어도 전체 콘텐츠 그룹이 상단에 몰려 있으면 올바른 중앙 배치로 보지 않는다.
- 여러 객체의 정렬이 깨졌을 경우 먼저 그룹 전체의 위치와 크기를 확인한 뒤 내부 간격을 조정한다.

## 2. Content Density Rules

PPT 슬라이드는 한 장에 하나의 핵심 메시지를 전달하는 것을 기본으로 한다.

정보를 많이 넣기보다 제목·요약·본문의 위계를 명확히 하여 빠르게 이해되는 보고서형 구성을 우선한다.

- 일반 본문 슬라이드는 하나의 핵심 메시지를 기준으로 구성한다.
- 본문 bullet은 3~5개를 권장한다.
- 카드가 적합한 병렬 정보는 3개 또는 4개 구성을 권장한다.
- 카드 사용 여부는 항목 수가 아니라 정보의 관계와 전달 목적을 기준으로 결정한다.
- 5개 이상의 병렬 항목은 표, 단계형 구조, 다이어그램 또는 별도 슬라이드로 분리한다.
- 제목, 긴 본문, 표, 카드 및 차트를 한 슬라이드에 모두 넣지 않는다.
- 글자 크기를 줄여 내용을 억지로 넣지 않는다.
- 슬라이드가 복잡해질 경우 내용을 무조건 줄이기보다 요약 슬라이드와 상세 슬라이드로 분리한다.
- 임원 보고 또는 고객 제안용 슬라이드는 제목과 요약만 읽어도 핵심 결론이 이해되도록 구성한다.
- 콘텐츠 양이 적은 경우 빈 공간을 카드나 장식으로 억지로 채우지 않고, 콘텐츠 그룹의 위치와 크기를 조정하여 균형을 만든다.
- 같은 유형의 카드 그리드를 여러 슬라이드에 연속해서 반복하지 않는다.
- 슬라이드의 내용 성격에 따라 표, 차트, 비교 구조, 프로세스, 다이어그램 또는 직접 배치형 레이아웃을 선택한다.
- 콘텐츠 양이 많을 경우 객체 간 간격을 지나치게 줄이기보다 슬라이드를 분리한다.

## 3. Approved Slide Layout Types

PPT 본문 슬라이드는 아래 레이아웃 중 하나를 우선 사용한다.

AI는 임의의 장식형 배치를 만들기보다 문서 목적과 콘텐츠 유형에 맞는 안정적인 보고서형 레이아웃을 선택한다.

모든 슬라이드를 카드 그리드로 통일하지 않는다. 카드, 표, 차트, 비교 구조, 프로세스, 다이어그램 및 직접 배치형 레이아웃 중 정보 성격에 가장 적합한 방식을 선택한다.

실제 레이아웃의 위치, 간격, 콘텐츠 크기 및 권장 문장 길이는 저장소의 PPT 레이아웃 참고자료를 함께 확인한다.

### 3.1 Title + Summary + Body

상단 제목, 핵심 요약, 본문 콘텐츠 순서로 구성하는 기본적인 보고서형 레이아웃이다.

- 상단에는 슬라이드 제목을 배치한다.
- 제목 아래에는 필요한 경우 핵심 요약문 또는 요약 카드를 배치한다.
- 본문 영역에는 bullet, 표, 카드 또는 차트 중 하나를 중심 콘텐츠로 배치한다.
- 보고서형 일반 본문 슬라이드에 우선 사용한다.
- 요약 영역과 본문 영역의 간격을 충분히 확보한다.
- 본문 콘텐츠 양이 적을 경우 요약문 바로 아래에 붙이지 않고, 남은 본문 영역의 세로 중심을 기준으로 배치한다.
- 요약문과 본문을 모두 상단에 몰아넣어 하단을 과도하게 비우지 않는다.

### 3.2 Two Column Layout

설명과 근거, 문제와 해결, 현황과 개선안처럼 대비되는 정보를 나란히 보여줄 때 사용한다.

- 좌측에는 배경, 설명 또는 문제 정의를 배치한다.
- 우측에는 수치, 결과, 표, 차트 또는 다이어그램을 배치한다.
- 좌우 컬럼의 시각적 무게가 균형을 이루도록 한다.
- 한쪽 컬럼에만 과도하게 많은 텍스트를 넣지 않는다.
- 좌우 컬럼 사이에는 충분한 간격을 둔다.
- 두 컬럼은 동일한 상단 기준선을 사용한다.
- 두 컬럼 전체를 하나의 콘텐츠 그룹으로 보고 본문 영역의 세로 중심을 기준으로 배치한다.
- 컬럼 높이가 낮을 경우 제목 바로 아래에 붙이지 않는다.

### 3.3 Card Grid Layout

핵심 항목, 서비스 영역, 추진 과제 및 기대 효과처럼 서로 독립적이고 병렬적인 정보를 정리할 때 사용한다.

카드 그리드는 PPT 본문의 기본 레이아웃이 아니라, 정보 성격이 카드에 적합한 경우에만 선택하는 레이아웃이다.

- 3개 또는 4개 카드 구성을 기본으로 한다.
- 모든 카드는 동일한 너비와 높이를 사용한다.
- 카드 내부의 제목, 설명 및 수치 위치를 통일한다.
- 카드 수가 많아지면 표 또는 별도 슬라이드로 분리한다.
- 카드마다 서로 다른 장식 요소를 넣지 않는다.
- 카드 간 간격은 균일하게 유지한다.
- 카드 그룹은 슬라이드의 가로 중앙뿐 아니라 본문 영역의 세로 중심도 고려하여 배치한다.
- 카드가 3개 또는 4개뿐인 슬라이드는 카드 그룹을 제목 바로 아래에 붙이지 않는다.
- 카드 그룹 위아래 여백이 시각적으로 균형을 이루도록 배치한다.
- 카드 높이가 낮을수록 그룹을 본문 영역 중앙에 가깝게 배치한다.
- 구성비, 추이, 순서, 관계형 데이터 및 긴 설명에는 카드 그리드를 우선 사용하지 않는다.
- 동일 문서에서 카드 그리드가 연속적으로 반복되지 않도록 다른 레이아웃과 조합한다.

### 3.4 Table Focus Layout

비교, 현황, 점검 결과, 일정 및 튜닝 결과처럼 표가 핵심 정보인 경우 사용한다.

- 표가 핵심인 슬라이드는 표를 본문 영역의 중심 콘텐츠로 배치한다.
- 표 위에는 표를 읽는 기준이나 핵심 요약을 한 줄로 작성할 수 있다.
- 표와 함께 카드, 차트 및 긴 본문을 동시에 배치하지 않는다.
- 표가 작아져 읽기 어려우면 슬라이드를 분리한다.
- 상세 표는 별도 슬라이드 또는 부록으로 분리할 수 있다.
- 행 수가 적고 표 높이가 낮은 경우 제목 아래에 붙이지 않고 본문 영역의 세로 중앙에 배치한다.
- 표 위 요약문이 있는 경우 요약문과 표를 하나의 그룹으로 보고 전체 그룹을 배치한다.

### 3.5 Chart Focus Layout

수치 변화, 추세, 비교 결과 및 구성비를 시각적으로 보여줄 때 사용한다.

- 차트가 핵심인 슬라이드는 차트를 가장 큰 정보 블록으로 배치한다.
- 차트 주변에는 핵심 인사이트 1~3개만 배치한다.
- 차트와 표를 같은 비중으로 나란히 배치하지 않는다.
- 차트 제목은 수치가 말하는 결론을 담아 작성한다.
- 차트 하단 또는 우측에는 필요한 경우 간단한 해석 문구를 배치한다.
- 차트와 해석 문구를 하나의 콘텐츠 그룹으로 보고 본문 영역의 중심을 기준으로 배치한다.
- 차트 높이가 낮다는 이유로 제목 바로 아래에 붙이지 않는다.

### 3.6 Direct Content Layout

긴 설명, 핵심 메시지, 정책, 배경 및 결론처럼 카드로 분리할 필요가 없는 내용을 직접 배치할 때 사용한다.

- 제목, 핵심 문장, 짧은 본문 및 필요한 보조 시각 요소를 직접 배치한다.
- 카드 테두리나 배경 박스가 없어도 여백, 정렬, 타이포그래피 위계로 정보 구조를 만든다.
- 하나의 중심 메시지를 크게 보여주고 보조 설명을 아래 또는 옆에 배치할 수 있다.
- 내용이 적다는 이유로 의미 없는 카드나 아이콘을 추가하지 않는다.
- 텍스트 블록은 본문 영역의 세로 균형을 고려하여 배치한다.
- 핵심 메시지와 설명의 위계를 크기, 굵기, 여백으로 구분한다.

### 3.7 Process or Step Layout

업무 흐름, 추진 단계, 적용 절차 및 로드맵처럼 순서가 있는 내용을 표현할 때 사용한다.

- 단계 수는 3~5개를 권장한다.
- 각 단계의 제목과 설명 길이를 비슷하게 유지한다.
- 단계 간 간격을 균일하게 유지한다.
- 연결선이나 화살표는 흐름을 보조하는 정도로 사용한다.
- 단계 그룹 전체를 본문 영역의 가로·세로 중심에 배치한다.
- 단계 수가 적을 경우 화면 상단에 몰리지 않도록 그룹의 폭과 위치를 함께 조정한다.

## 4. Text Overflow & Wrapping Rules

PPT는 화면 크기가 제한되어 있으므로 한글 문장의 줄바꿈과 텍스트 넘침을 반드시 고려한다.

- 제목은 1~2줄 안에서 작성한다.
- 제목이 2줄을 초과하면 문장을 줄이거나 부제 또는 본문으로 분리한다.
- 카드 제목은 1줄을 기본으로 하며 최대 2줄을 넘지 않는다.
- 카드 설명은 2~3줄 이내로 작성한다.
- 본문 bullet은 한 항목당 1~2줄을 권장한다.
- 표 셀 안의 문장은 줄바꿈을 고려하여 충분한 행 높이를 확보한다.
- 텍스트가 박스 밖으로 넘치지 않도록 텍스트 박스 너비를 우선 확보한다.
- 글자 크기를 과도하게 줄여 해결하지 않는다.
- 긴 문장은 핵심어와 설명을 분리하여 작성한다.
- 한글과 영문이 섞인 문장은 줄바꿈 위치가 어색하지 않도록 문장 길이를 조정한다.
- 수치, 단위, 제품명 및 고객사명은 줄바꿈으로 분리되어 의미가 깨지지 않도록 한다.
- 텍스트를 줄인 뒤 콘텐츠 높이가 낮아졌다면 본문 그룹의 세로 위치도 다시 조정한다.
- 텍스트 양이 줄었는데 기존 상단 위치를 그대로 유지하여 하단 여백이 과도하게 남지 않도록 한다.

## 5. PPT Table Stability Rules

표 정렬 기준은 저장소의 공통 디자인 기준을 따른다.

이 섹션에서는 PPT 슬라이드 안에서 표가 깨지거나 읽기 어려워지는 문제를 방지한다.

- 표는 슬라이드 중앙 안전 영역 안에 배치한다.
- 표가 슬라이드 가장자리까지 밀리지 않도록 한다.
- 긴 설명 컬럼은 너비를 넓게 배정하고 수치 컬럼은 좁게 배정한다.
- 표 안의 텍스트가 셀 밖으로 넘치지 않도록 행 높이를 확보한다.
- 표가 8행 또는 6열을 초과하면 요약 표와 상세 표를 분리한다.
- 표가 작아져 읽기 어려운 경우 별도 상세 슬라이드로 분리한다.
- 표 상단에는 표를 해석하는 기준이나 핵심 요약을 짧게 배치할 수 있다.
- 표 하단에는 필요한 경우 단위, 기준일, 출처 및 주석을 작게 배치한다.
- 표 안에 지나치게 많은 색상 강조를 넣지 않는다.
- 강조가 필요한 행이나 셀은 1~3개 이내로 제한한다.
- 표, 상단 요약 및 하단 주석을 하나의 콘텐츠 그룹으로 보고 전체 위치를 결정한다.
- 표의 행 수가 적어 전체 높이가 낮은 경우 본문 영역 상단에 붙이지 않고 세로 중심을 기준으로 배치한다.

## 6. Reference Usage Rules

PPT 제작 시 텍스트 규칙만으로 판단하기 어려운 요소는 저장소의 PPT 참고자료를 함께 확인한다.

- 참고자료에서는 제목 위치, 본문 시작 위치, 카드 간격, 표 크기, 차트 비율, 하단 여백 및 콘텐츠 그룹의 세로 중심을 확인한다.
- 참고자료의 문구는 실제 작성 내용이 아니라 레이아웃의 크기와 밀도를 확인하기 위한 예시로 본다.
- 참고자료를 슬라이드 내용 작성용 원본으로 오해하지 않는다.
- 참고자료의 레이아웃을 그대로 복제할 필요는 없으나 정렬, 여백 및 콘텐츠 밀도 기준은 유지한다.
- 참고자료와 본 가이드가 충돌할 경우 본 가이드의 명시적 규칙을 우선한다.
- 참고자료를 정상적으로 읽지 못한 경우에도 본 가이드의 제목·본문 영역 분리 및 세로 균형 규칙을 반드시 적용한다.
- 특정 참고자료 파일명에 의존하지 않고, PPT 레이아웃과 디자인에 해당하는 자료를 탐색하여 활용한다.

## 7. PPT Output Review Checklist

PPT 산출 전 아래 항목을 확인한다.

### Layout

- [ ] 기본 슬라이드 비율이 16:9로 구성되었는가?
- [ ] 제목 영역과 본문 콘텐츠 영역이 구분되어 있는가?
- [ ] 핵심 콘텐츠가 슬라이드 중앙 안전 영역 안에 배치되었는가?
- [ ] 본문 콘텐츠가 좌측 또는 우측으로 의도 없이 치우치지 않았는가?
- [ ] 제목, 카드, 표 및 차트의 좌우 기준선이 정리되어 있는가?
- [ ] 개별 객체가 아니라 콘텐츠 그룹 전체를 기준으로 정렬했는가?

### Vertical Balance

- [ ] 콘텐츠 양이 적은데도 제목 바로 아래에 붙여 배치하지 않았는가?
- [ ] 본문 콘텐츠 그룹이 제목을 제외한 본문 영역의 세로 중심에 배치되었는가?
- [ ] 콘텐츠 그룹의 상단 여백과 하단 여백이 시각적으로 균형을 이루는가?
- [ ] 슬라이드 하단에 과도한 빈 공간이 남지 않았는가?
- [ ] 카드 3개 또는 4개만 있는 슬라이드가 화면 상단에 몰리지 않았는가?
- [ ] 표나 차트의 높이가 낮을 때 본문 영역 중앙으로 위치를 조정했는가?
- [ ] 내용 수정으로 콘텐츠 높이가 달라진 뒤 위치를 다시 검토했는가?

### Content Density

- [ ] 한 슬라이드에 하나의 핵심 메시지만 담겼는가?
- [ ] 본문 슬라이드를 일률적인 카드 나열형으로 구성하지 않았는가?
- [ ] 카드가 정보 성격에 적합한 경우에만 사용되었는가?
- [ ] 같은 3개 또는 4개 카드 구조가 여러 슬라이드에 반복되지 않았는가?
- [ ] 표, 차트, 비교 구조, 프로세스, 다이어그램 또는 직접 배치형 레이아웃을 함께 검토했는가?
- [ ] 본문 bullet이 과도하게 많지 않은가?
- [ ] 카드 수가 3개 또는 4개 기준을 크게 벗어나지 않는가?
