# VATOS Document Theme Guide

## 1. Purpose

본 문서는 VATOS 문서 작성 시 공통으로 적용할 표지, 내지, 마무리 페이지의 기본 테마 선택 및 적용 기준을 정의한다.

VATOS의 모든 문서는 저장소의 공통 디자인 기준을 기반으로 하며, 별도 디자인 요청이 없는 경우 본 문서의 기본 테마 원칙을 우선 적용한다.

본 가이드는 Word, PowerPoint, Excel, HTML, Markdown 문서에 공통으로 적용할 수 있는 시각적 방향을 제공한다. 문서 유형별 세부 레이아웃, 출력 안정성, 페이지 구성 및 편집 규칙은 해당 문서 유형 가이드를 함께 적용한다.

---

## 2. Theme Reference Usage

테마의 실제 색감, 표지 구성, 내지 분위기, 기하 그래픽, 머리말·바닥글 표현은 저장소의 테마 참고자료와 완성 예시를 함께 확인한다.

- 테마 참고자료는 색상 조합과 시각적 분위기를 확인하기 위한 자료로 사용한다.
- 완성 예시는 실제 산출물의 밀도, 구성, 정보 위계 및 마감 수준을 확인하기 위한 품질 기준으로 사용한다.
- 승인된 예시는 고정 템플릿이 아니며, 제목 위치·도형 비율·면 분할·사선 위치를 그대로 반복 복제하지 않는다.
- 템플릿은 실제 문서 작성을 시작하는 편집 가능한 원본으로 사용한다.
- 특정 파일명이나 경로에 의존하지 않고, 역할이 일치하는 자료를 찾아 활용한다.
- 참고자료는 색상 면적 비율, 깊이감, 여백, 타이포그래피 위계와 전문성을 판단하는 기준으로 활용한다.
- 참고자료와 본 가이드가 충돌할 경우 본 가이드의 명시적 규칙을 우선한다.

## 3. Theme Concept

VATOS 문서 테마는 다음 인상을 기준으로 한다.

- 도회적인
- 신뢰감 있는
- 전문적인
- 정돈된
- 기술 중심의
- 과하지 않은

기본 문서 분위기는 Urban Navy 기반의 표지와 밝은 내지 조합으로 구성한다.

```text
Cover / Closing
→ Urban Navy background + White title + Cyan accent

Internal Page
→ Light surface + Navy accent + Thin border + Structured layout
```

Urban Navy는 브랜드 인상을 형성하는 기본 테마이며, 모든 페이지를 어둡게 구성하는 다크 문서 테마를 의미하지 않는다.

VATOS에서 **Urban**은 도시 사진, 건축물, 도로, 야경 또는 스카이라인을 의미하지 않는다. Urban은 정돈된 비대칭 구성, 깊은 네이비 명암, 구조적인 기하 표현, 현대적인 여백과 타이포그래피를 의미한다.

VATOS 테마는 특정 표지 레이아웃 하나를 반복하는 고정 템플릿이 아니다. 브랜드 컬러 역할과 시각적 품질은 유지하되, 문서 목적·대상 독자·제목 길이·정보량에 따라 구도와 조형 방식을 다양하게 구성한다.

---

## 4. Theme Usage Overview

| Document Area | Theme Usage | Design Rule |
|---|---|---|
| Cover | Urban Navy 중심 | 네이비 배경, 흰색 타이틀, Cyan 포인트, 문서 목적에 맞는 제한적인 기하 구성 |
| Internal | Light surface + Navy accent | 밝은 배경, 네이비 제목·라인, 얇은 보더와 정돈된 정보 구조 |
| Closing | Urban Navy 중심 | 표지와 동일한 시각 언어를 공유하되 동일 구도는 반복하지 않음 |
| Table / Card | Light surface | 얇은 보더, 충분한 여백, 정보 중심 구성 |
| Footer / Header | Minimal guide line | 옅은 구분선, 문서 정보, 페이지 번호 또는 슬로건 |

## 5. Cover Theme

표지는 문서의 첫인상을 결정하는 영역으로, VATOS의 전문성과 기술 중심 이미지를 전달해야 한다.

### 5.1 Layout

표지는 다음 요소를 기본으로 구성한다.

```text
Logo
Document Type
Document Title
Subtitle
Footer Information
Decorative Graphic
```

### 5.2 Cover Placeholder Text

표지 템플릿에는 실제 문구 대신 아래의 입력 가이드 문구를 사용할 수 있다.

```text
문서 타입을 입력하세요
예) 회사소개서 · 작업계획서 · 점검 리포트

문서 제목을 입력하세요

문서 부제를 입력하세요.
문서의 목적, 핵심 요약, 적용 대상 등을 한두 문장으로 작성하세요.
```

### 5.3 Cover Design Rules

- 배경은 Urban Navy 계열을 중심으로 사용한다.
- Premium Dark 또는 네이비 계열의 명도 차이로 깊이감을 만들 수 있다.
- 깊이 표현 시 네이비 면을 딱 끊어 분리하기보다 Urban Navy에서 더 어두운 네이비로 자연스럽게 이어지는 절제된 그라데이션을 우선한다.
- 그라데이션은 동일 네이비 계열의 명도 차이 안에서만 사용하며, 다색·고채도·광택형 그라데이션은 사용하지 않는다.
- 문서 제목은 White 계열로 크게 배치하고, 문서에서 가장 먼저 인식되도록 한다.
- 문서 타입, 섹션 라벨 또는 보조 문구에는 Cyan을 제한적인 포인트로 사용한다.
- Cyan은 넓은 배경이 아니라 얇은 선, 짧은 구분선, 작은 라벨 또는 핵심 포인트로 사용한다.
- 제목, 부제, 문서 유형, 작성 정보의 위계를 명확하게 구분한다.
- 텍스트 영역과 그래픽 영역 사이에 충분한 여백을 확보한다.
- 정중앙 대칭보다 의도된 비대칭 구성을 사용할 수 있다.
- 그래픽 요소는 제목을 방해하지 않는 보조 요소로 사용한다.
- 한 표지에서는 하나의 주요 조형 방식을 선택한다.
- 표지 전체를 일반 카드 컴포넌트처럼 구성하지 않는다.
- 하단에는 회사명, 웹사이트 또는 슬로건을 배치할 수 있다.
- Cover와 Closing의 하단 정보 영역을 사용할 경우, 본문과 구분되도록 Soft Gray 또는 White 저투명도의 1px 구분선을 적용할 수 있다.
- 하단 구분선은 전체 폭을 과도하게 강조하지 않고 안전 영역 안에서 얇고 절제되게 사용한다.
- 표지 요소는 템플릿이나 문서 유형에 따라 생략할 수 있으나, 로고와 문서 제목의 위계는 명확히 유지한다.
- 표지 전체를 통이미지로 만들지 않고 핵심 텍스트는 편집 가능한 상태로 유지한다.

### 5.4 Cover Visual Language

표지에는 아래 조형 방식 중 문서 목적에 적합한 한 가지를 중심으로 사용할 수 있다.

| Motif | Description | Recommended Use |
|---|---|---|
| Diagonal Layer | 네이비 명도 차이와 사선 면을 활용한 구성 | 회사소개서, 제안서, 주요 발표자료 |
| Vertical Band | 세로 밴드 또는 측면 컬러 영역을 활용한 구성 | 기술 보고서, 업무 보고서 |
| Offset Frame | 얇은 프레임과 넓은 여백을 활용한 구성 | 임원 보고, 공식 문서 |
| Tonal Block | 네이비 계열 면을 크고 단순하게 분할한 구성 | 프로젝트 보고서, 전략 문서 |
| Thin Line Grid | 얇은 선과 구조적인 그리드를 활용한 구성 | 데이터, DBMS, 시스템 문서 |
| Typography Focus | 제목과 여백 중심으로 장식을 최소화한 구성 | 간결한 보고서, 내부 문서 |
| Cropped Geometry | 화면 밖으로 잘린 추상 기하 면을 활용한 구성 | 회사소개서, 브랜드 문서 |

#### Motif Usage Rules

- 한 표지에는 주요 Motif를 하나만 선택한다.
- 보조 요소는 주요 Motif를 강화하는 범위에서만 사용한다.
- 사선 레이어는 선택 가능한 방식 중 하나이며 필수 요소가 아니다.
- 승인 예시의 사선 구성을 모든 표지에 반복하지 않는다.
- 동일한 컬러를 사용하더라도 문서마다 구도, 면 분할, 선 방향 및 그래픽 위치를 달리할 수 있다.
- 기하 요소는 의미 없는 장식이 아니라 화면 균형과 정보 위계를 만드는 목적으로 사용한다.
- 사선, 원, 그리드, 프레임 등 서로 다른 Motif를 한 화면에 과도하게 혼합하지 않는다.

### 5.5 Theme Variation Rules

- 승인 예시는 고정 템플릿이 아니라 컬러 균형, 깊이감, 여백, 타이포그래피 위계와 전문성의 품질 기준이다.
- 승인 예시와 동일한 구도를 자동으로 반복하지 않는다.
- 문서 목적에 따라 제목 위치, 면 분할, 선 방향, 그래픽 위치를 변경할 수 있다.
- Cover, Section Divider, Closing은 동일한 시각 언어를 공유하되 동일한 구도를 그대로 복제하지 않는다.
- 동일 문서 안에서는 선택한 조형 언어를 일관되게 유지한다.
- 서로 다른 문서에서는 기존 산출물과 구도가 과도하게 반복되지 않도록 한다.
- 새로운 구도를 사용할 때도 승인 예시와 동등한 수준의 정보 위계와 마감 품질을 확보한다.

### 5.6 Cover Footer Example



```text
주식회사 바토스 · VATOS Co., Ltd.        vatos.kr · inspire the next
```

## 6. Internal Page Theme

내지는 문서의 본문을 담는 영역이므로 가독성과 정보 구조를 가장 우선한다.

### 6.1 Layout

내지 페이지는 다음 구성을 기본으로 한다.

```text
Header
Title / Section Title
Body Content
Table / Card / Figure
Footer
```

### 6.2 Header

본문 페이지 상단에는 문서 정보를 나타내는 머리말을 적용할 수 있다.

```text
좌측: 로고 또는 문서명
우측: VATOS 또는 슬로건
하단: 옅은 회색 구분선
```

예시:

```text
VA  COMPANY PROFILE                         VATOS · inspire the next
────────────────────────────────────────────────────────────
```

### 6.3 Footer

본문 페이지 하단에는 바닥글을 적용할 수 있다.

```text
좌측: 회사명 또는 문서명
우측: 페이지 번호
상단: 옅은 회색 구분선
```

예시:

```text
────────────────────────────────────────────────────────────
주식회사 바토스 · Company Profile 2026                    02
```

### 6.4 Internal Page Design Rules

- 배경은 White 또는 Light Surface를 사용한다.
- 제목은 Navy 또는 Charcoal 계열로 강조한다.
- 섹션 구분에는 얇은 라인, 여백 또는 제한적인 배경 차이를 사용한다.
- 본문은 과도한 장식 없이 간결하게 구성한다.
- 표와 카드는 얇은 보더와 충분한 여백을 사용한다.
- 강조가 필요한 경우 Cyan 포인트를 제한적으로 사용한다.
- 한 페이지에 너무 많은 카드나 장식 요소를 넣지 않는다.
- 본문 전체를 카드 나열형으로만 구성하지 않는다.
- 카드가 필요하지 않은 정보는 제목, 본문, 표, 차트, 다이어그램을 직접 배치하는 레이아웃을 사용한다.
- 머리말과 바닥글은 본문 영역을 침범하지 않도록 한다.
- 정보량이 많은 페이지는 다크 배경보다 밝은 배경을 우선한다.

## 7. Closing Theme

마무리 페이지는 문서의 종료를 명확하게 전달하는 영역이다.

### 7.1 Layout

마무리 페이지는 다음 요소를 기본으로 구성한다.

```text
Logo
Thank You
Closing Message
Company Information
Web / Tagline
Decorative Graphic
```

### 7.2 Closing Placeholder Text

```text
Thank You

마무리 문구를 입력하세요.
본 문서의 핵심 메시지 또는 전달하고 싶은 마무리 문장을 작성하세요.
```

### 7.3 Closing Design Rules

- 표지와 연결되는 Urban Navy 배경을 사용한다.
- Thank You 문구는 White 계열로 크게 배치할 수 있다.
- 마무리 문구는 한두 줄 이내로 작성한다.
- 하단에는 회사 정보, 주소, 웹사이트 또는 슬로건을 배치할 수 있다.
- 표지에서 선택한 조형 언어를 단순화하거나 재배치하여 문서의 시작과 끝이 연결되도록 한다.
- 표지의 구도를 그대로 축소하거나 복제하지 않는다.
- 회사 정보는 최신 정보인지 최종 검수한다.
- 모든 문서 유형에 Closing을 의무적으로 사용하지 않는다.

## 8. Color Usage

세부 컬러 값과 토큰은 저장소의 공통 디자인 기준을 따른다. 본 문서에서는 테마 선택과 적용 역할만 정의한다.

| Role | Color Group | Usage |
|---|---|---|
| Primary | Urban Navy | 표지 배경, 주요 헤더, 브랜드 강조 영역 |
| Accent | Cyan | 포인트 라인, 섹션 라벨, 제한적인 강조 요소 |
| Surface | White / Light Surface | 본문 배경, 카드 배경, 표 본문 |
| Text Primary | Navy / Charcoal | 제목, 본문, 주요 텍스트 |
| Text Secondary | Gray | 설명 문구, 보조 정보, 주석 |
| Border | Light Gray | 표, 카드, 머리말·바닥글 구분선 |

### 8.1 Color Rules

- 표지와 마무리 페이지는 Urban Navy를 중심으로 구성한다.
- 내지 페이지는 밝은 배경을 기본으로 한다.
- Cyan은 브랜드 강조용으로만 사용하며 넓은 면적에 남용하지 않는다.
- Cyan을 성공, 개선, 완료 등 상태 의미로 사용하지 않는다.
- 회색 라인은 구분선과 보조 정보 영역에만 사용한다.
- 배경과 텍스트의 대비를 충분히 확보한다.
- 모든 산출물을 다크 테마로 구성하지 않는다.

## 9. Typography and Alignment

세부 서체, 크기 및 계층 기준은 저장소의 공통 디자인 기준을 따른다.

### 9.1 Heading

- 문서 제목은 크고 명확하게 배치한다.
- 섹션 제목은 Navy 또는 Charcoal 계열을 사용한다.
- 부제와 설명 문구는 본문보다 작고 가볍게 표현한다.
- 제목은 단순한 항목명이 아니라 페이지의 핵심 메시지를 전달하도록 작성한다.

### 9.2 Body

- 본문은 읽기 쉬운 크기와 줄 간격을 유지한다.
- 긴 문장은 피하고 문단을 짧게 나눈다.
- 표와 카드 안의 텍스트는 정렬 기준을 통일한다.
- 하나의 문서에서 여러 서체를 혼용하지 않는다.

### 9.3 Alignment

| Content Type | Alignment |
|---|---|
| 제목 | Left |
| 본문 | Left |
| 긴 설명 | Left |
| 수치 | Right |
| 표 헤더 | Center |
| 짧은 상태값 | Center |

---

## 10. Table and Card Style

### 10.1 Table

표는 정보 전달을 목적으로 하며 장식적인 요소를 최소화한다.

- 헤더는 밝은 배경 또는 제한적인 Navy Accent를 사용한다.
- 행 간 구분선은 얇게 적용한다.
- 수치 값은 오른쪽 정렬한다.
- 설명이 긴 항목은 왼쪽 정렬한다.
- 표 안의 여백은 충분히 확보한다.
- 상태 정보는 의미에 맞는 Semantic Color를 사용한다.
- 표가 문서 유형별 출력 영역을 넘어가지 않도록 해당 가이드의 안정성 기준을 함께 적용한다.

### 10.2 Card

카드는 핵심 정보 요약에 사용한다.

- 카드 배경은 White 또는 Light Surface를 사용한다.
- 보더는 얇게 적용한다.
- 그림자는 최소화한다.
- 한 페이지에 카드가 과도하게 많아지지 않도록 한다.
- 카드 하나에는 하나의 메시지만 담는다.
- 카드 수와 배치는 문서 유형별 레이아웃 기준을 따른다.

## 11. Document Type Usage

문서 유형별 권장 사용 방식은 아래와 같다.

| Document Type | Cover | Internal Page | Closing | Note |
|---|---|---|---|---|
| Company Profile | Required | Required | Required | 외부 공유 전 회사 정보와 연락처 검수 |
| Proposal | Required | Required | Required | 고객사명, 프로젝트명 및 제안 범위 확인 |
| Work Plan | Required | Required | Optional | 작업 범위와 일정 중심 |
| Work Result | Required | Required | Optional | 결과와 증빙 중심 |
| Inspection Report | Optional | Required | Optional | 표와 수치 가독성 우선 |
| Tuning Report | Optional | Required | Optional | Before / After 비교와 근거 강조 |
| Internal Manual | Optional | Required | Optional | 검색성과 유지보수 우선 |
| Onboarding Guide | Optional | Required | Optional | 단계별 안내 중심 |

### 11.1 PowerPoint

- 표지, 섹션 구분 및 Closing에는 Urban Navy를 사용할 수 있다.
- 정보량이 많은 본문 슬라이드는 밝은 배경을 우선한다.
- 실제 요소 위치와 간격은 PPT 제작 가이드와 레이아웃 참고자료를 함께 확인한다.

### 11.2 Word

- 표지는 Urban Navy 테마를 적용할 수 있다.
- 내지는 밝은 배경과 최소한의 머리말·바닥글을 사용한다.
- 표, 그림 및 페이지 흐름은 Word 제작 가이드의 안정성 규칙을 적용한다.

### 11.3 Excel

- 표지는 문서 목적에 따라 선택적으로 사용한다.
- 데이터 시트는 장식보다 가독성, 필터, 정렬 및 데이터 입력 편의성을 우선한다.
- Urban Navy는 제목 영역이나 구분용 헤더에 제한적으로 사용한다.
- 데이터 영역 전체에 어두운 배경을 사용하지 않는다.

### 11.4 HTML

- 표지 또는 히어로 영역에는 Urban Navy를 사용할 수 있다.
- 긴 본문, 표, 카드 및 검색 영역은 밝은 배경을 우선한다.
- 외부 자산에 의존하지 않는 정적 문서의 경우에도 동일한 테마 역할을 유지한다.

### 11.5 Markdown

- Markdown 문법만으로 색상과 배경을 완전히 재현하기 어려운 경우 정보 구조와 제목 계층을 우선한다.
- 로고나 이미지가 필요한 경우 저장소의 승인된 브랜드 자산을 상대 경로로 참조한다.
- HTML이나 CSS를 사용할 수 없는 환경에서는 테마를 장식이 아닌 문서 구조로 표현한다.

## 12. Template, Reference and Example Usage

실제 문서 작성 시에는 아래 원칙을 따른다.

```text
1. 산출물 유형에 맞는 문서 가이드를 확인한다.
2. 편집 가능한 템플릿이 있으면 작업 원본으로 사용한다.
3. 테마 참고자료에서 색감과 시각적 분위기를 확인한다.
4. 완성 예시에서 구성과 마감 수준을 확인한다.
5. 표지의 문서 타입, 제목, 부제를 수정한다.
6. 머리말과 바닥글의 문서 정보를 수정한다.
7. 최종 배포 전 오탈자, 고객사명, 날짜, 버전 및 회사 정보를 검수한다.
```

- 템플릿은 실제 작업을 시작하는 편집 가능 원본이다.
- 참고자료는 디자인, 배치 또는 테마 판단을 돕는 자료이다.
- 예시는 디자인 시스템이 적용된 완성 결과물이며 동일 유형의 문서 작성 시 구성과 마감 수준을 판단하는 품질 기준으로 활용한다.
- 예시의 제목 위치, 도형 비율, 사선 위치 및 면 분할을 그대로 복제하지 않는다.
- 예시에서 컬러 면적 비율, 네이비 깊이감, Cyan 사용량, 여백, 타이포그래피 위계와 전문성을 우선 참고한다.
- 동일 파일을 여러 폴더에 중복 저장하지 않는다.
- 자료의 실제 주제와 목적은 파일명과 내용으로 확인하되, 가이드 문서에서는 특정 파일명을 직접 고정하지 않는다.

## 13. Do

VATOS 문서 테마를 적용할 때 아래 원칙을 지킨다.

- 표지는 Urban Navy 기반으로 구성하되 특정 표지 구도를 고정 템플릿처럼 반복하지 않는다.
- 문서 제목은 명확하게 크게 배치한다.
- Cyan 포인트는 제한적으로 사용한다.
- 본문은 밝은 배경과 충분한 여백을 사용한다.
- 표와 카드는 얇은 보더 중심으로 구성한다.
- 필요한 경우 머리말과 바닥글에 문서 정보와 페이지 번호를 표시한다.
- 문서 유형에 맞는 가이드, 템플릿, 참고자료 및 완성 예시를 함께 확인한다.
- 한 표지에는 주요 조형 방식 하나만 선택한다.
- Cover, Section Divider, Closing은 같은 시각 언어를 공유하되 서로 다른 구도로 구성한다.
- 최종 배포 전 오탈자, 고객사명, 날짜, 버전 및 회사 정보를 확인한다.
- 로고와 원본 이미지를 제외한 요소는 가능한 한 편집 가능한 형태로 유지한다.

---

## 14. Don’t

아래 표현은 사용하지 않는다.

- 과도한 그라데이션
- 과한 네온 효과
- 두꺼운 그림자
- 불필요한 3D 오브젝트
- 의미 없는 이모지 장식
- 지나치게 많은 카드 나열
- 본문 전체를 어두운 배경으로 구성
- 낮은 대비의 텍스트
- Cyan을 상태 의미로 사용
- 고객사명, 수치, 정책 또는 회사 정보의 임의 생성
- 표지와 내지의 톤이 완전히 다른 구성
- 동일 예시 파일을 여러 폴더에 중복 저장
- 특정 파일명이나 경로에만 의존하는 안내 문구
- 도회적이라는 표현을 도시 사진, 건축물, 도로, 야경 또는 스카이라인 이미지로 해석
- 네이비 직사각형 하나만 배치하여 표지를 완성
- Dark Theme와 Light Theme를 단순한 색상 견본 카드처럼 표현
- 표지의 핵심 시각 요소를 일반 카드 컴포넌트로 대체
- 승인된 표지 예시의 구도를 모든 문서에 동일하게 복제
- 모든 표지에 사선 레이어를 의무적으로 사용
- 한 표지에서 여러 종류의 기하 Motif를 과도하게 혼합
- Cyan을 넓은 배경 면적으로 사용
- 의미 없는 원형 장식, 네온 광선, 도시 실루엣 또는 복잡한 배경 패턴

## 15. Relationship with Other Resources

본 문서는 VATOS 문서 작성의 기본 테마 선택과 적용 기준을 정의한다.

```text
common-design-rules
→ 전체 산출물의 컬러, 타이포그래피, 컴포넌트 및 브랜드 공통 기준

theme-usage-rules
→ 표지, 내지, Closing의 기본 테마 적용 기준

document-type-guides
→ PPT, Word 등 산출물 유형별 제작 및 안정성 기준

document-templates
→ 실제 문서 작성에 사용하는 편집 가능한 원본

document-examples
→ 디자인 시스템이 적용된 완성 결과물

brand-assets
→ 공식 로고 및 테마 관련 브랜드 자산
```

특정 파일명이나 경로는 이 문서에 고정하지 않는다. 실제 구조와 자료 목록은 저장소 안내 문서에서 관리한다.

## 16. Review Checklist

### Theme Selection

- [ ] 별도 요청이 없는 경우 Urban Navy 표지와 밝은 내지 조합을 적용했는가?
- [ ] 문서 목적과 정보량에 맞는 테마를 선택했는가?
- [ ] 모든 페이지를 어두운 배경으로 구성하지 않았는가?

### Color and Contrast

- [ ] Cyan을 제한적인 브랜드 강조로 사용했는가?
- [ ] Cyan을 성공이나 개선 의미로 사용하지 않았는가?
- [ ] 배경과 텍스트의 대비가 충분한가?
- [ ] 본문과 표는 밝은 배경에서 읽기 쉽게 구성했는가?

### Structure

- [ ] 표지, 내지 및 마무리 페이지의 톤이 연결되는가?
- [ ] Cover, Section Divider, Closing이 같은 시각 언어를 공유하되 동일 구도를 반복하지 않았는가?
- [ ] 승인 예시의 구도를 그대로 복제하지 않았는가?
- [ ] 한 표지에서 주요 Motif가 하나로 정리되었는가?
- [ ] Urban을 도시 이미지 또는 스카이라인으로 해석하지 않았는가?
- [ ] 머리말과 바닥글이 본문을 침범하지 않는가?
- [ ] 표와 카드가 과도하게 장식적이지 않은가?
- [ ] 본문 전체가 단순 카드 나열형으로만 구성되지 않았는가?
- [ ] 문서 유형별 가이드를 함께 적용했는가?

### Resources and Output

- [ ] 적절한 템플릿, 참고자료 또는 완성 예시를 확인했는가?
- [ ] 특정 파일명이나 경로에만 의존하지 않았는가?
- [ ] 승인된 브랜드 자산만 사용했는가?
- [ ] 로고와 원본 이미지를 제외한 요소가 수정 가능한 형태인가?
- [ ] 고객사명, 날짜, 버전 및 회사 정보를 최종 검수했는가?
