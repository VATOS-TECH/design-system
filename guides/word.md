# VATOS Word Design Guide

## Purpose

이 문서는 VATOS 공통 디자인 기준을 기반으로, Word 또는 일반 문서 산출물에서 발생하기 쉬운 페이지 흐름 불안정, 표 깨짐, 불필요한 빈 페이지, 긴 본문 가독성 저하 및 편집 과정의 레이아웃 변형을 방지하기 위한 문서 유형별 세부 규격이다.

공통 브랜드 컬러, 로고, 타이포그래피, 디자인 톤, Do/Don't 및 수정 가능성 원칙은 저장소의 공통 디자인 기준을 따른다.

Word 작업 시 본 가이드의 규칙을 우선 적용하고, 표지와 문서 형식은 저장소의 Word 템플릿 자료를 함께 확인한다.

---

## 1. Page Setup Rules

- 기본 용지는 A4를 기준으로 한다.
- 문서 방향은 세로를 기본으로 하며, 넓은 표나 도표가 필요한 경우에만 해당 구간에 가로 방향을 적용한다.
- 기본 여백을 기준으로 제목, 본문, 표, 그림 및 주석이 인쇄 영역 안에 안정적으로 배치되도록 한다.
- 본문 요소를 페이지 가장자리까지 확장하지 않는다.
- 표지, 본문, 부록의 페이지 구성을 명확히 구분한다.
- 구역 나누기는 페이지 방향, 머리말·바닥글 또는 번호 체계가 달라지는 경우에만 사용한다.
- 불필요한 구역 나누기와 페이지 나누기를 반복해서 사용하지 않는다.

---

## 2. Document Structure Rules

- 문서는 표지, 개정 이력, 목차, 본문, 부록 순서를 기본으로 구성한다.
- 문서 목적에 따라 불필요한 섹션은 생략할 수 있다.
- 제목 계층은 일관되게 유지한다.
- 제목 수준을 건너뛰지 않는다.
- 본문은 긴 문단보다 짧은 문단과 목록을 우선 사용한다.
- 한 문단에는 하나의 중심 내용을 담는다.
- 번호 체계는 문서 전체에서 동일한 형식을 유지한다.
- 페이지가 바뀌어도 제목과 본문이 자연스럽게 이어지도록 한다.
- 페이지 하단에 제목만 단독으로 남지 않도록 다음 본문과 함께 배치한다.

---

## 3. Paragraph and Spacing Rules

- 문단 간격은 빈 문단을 반복해서 넣는 방식으로 만들지 않는다.
- 제목 앞뒤, 본문 문단, 목록 및 표 사이의 간격은 문단 서식으로 관리한다.
- 줄 간격과 문단 간격을 혼용하여 과도한 공백을 만들지 않는다.
- 본문 정렬은 왼쪽 정렬을 기본으로 한다.
- 긴 본문에 강제 줄바꿈을 반복해서 사용하지 않는다.
- 강조가 필요한 문장은 굵기 또는 제한된 강조 색상으로 표현한다.
- 동일한 문서 안에서 제목 크기와 본문 크기를 임의로 반복 변경하지 않는다.

---

## 4. Table Stability Rules

표 정렬과 색상 기준은 저장소의 공통 디자인 기준을 따른다.

- 표는 페이지의 인쇄 가능 영역 안에 배치한다.
- 표가 페이지 폭을 넘어가지 않도록 전체 너비와 컬럼 너비를 조정한다.
- 긴 설명 컬럼은 넓게 배정하고, 수치·상태·구분 컬럼은 상대적으로 좁게 배정한다.
- 표 안에 긴 문장과 다중 bullet을 과도하게 넣지 않는다.
- 표 셀의 텍스트가 잘리거나 겹치지 않도록 행 높이와 셀 여백을 확보한다.
- 한 행이 페이지 사이에서 분리되어 읽기 어려워지지 않도록 설정한다.
- 여러 페이지로 이어지는 표는 머리글 행을 반복한다.
- 표가 한 페이지에서 지나치게 깨질 경우 요약 표와 상세 표로 분리한다.
- 표 전체를 한 페이지에 억지로 맞추기 위해 글자 크기나 셀 여백을 과도하게 줄이지 않는다.
- 넓은 표는 해당 구간만 가로 방향으로 전환하거나 부록으로 분리한다.
- 표 전후에는 본문과 구분되는 적절한 간격을 둔다.
- 표 하단에는 필요한 경우 단위, 기준일, 출처 및 주석을 배치한다.

---

## 5. Blank Page Prevention Rules

불필요한 빈 페이지는 대부분 과도한 나누기, 빈 문단, 표 뒤의 잔여 문단 또는 구역 설정으로 인해 발생한다.

- 빈 페이지를 만들기 위해 Enter 키를 반복하지 않는다.
- 새 페이지가 필요한 경우 페이지 나누기를 사용한다.
- 페이지 나누기와 구역 나누기를 중복해서 사용하지 않는다.
- 표 뒤에 자동 생성되는 마지막 문단이 다음 페이지로 밀리지 않도록 문단 간격과 글자 크기를 확인한다.
- 숨겨진 빈 문단과 수동 줄바꿈이 누적되지 않았는지 확인한다.
- 제목의 `다음 문단과 함께`, 본문의 `문단 나누지 않음` 설정이 과도하게 적용되지 않았는지 확인한다.
- 구역 나누기 종류가 `다음 페이지`, `홀수 페이지`, `짝수 페이지` 중 의도와 맞는지 확인한다.
- 페이지 방향을 변경한 뒤 불필요한 빈 구역이나 빈 페이지가 남지 않았는지 확인한다.
- 내용이 없는 마지막 페이지가 생성되면 마지막 표, 문단, 구역 나누기 및 머리말·바닥글 연결 상태를 우선 점검한다.

---

## 6. Header, Footer and Page Number Rules

- 머리말과 바닥글은 본문 영역을 침범하지 않도록 충분한 간격을 둔다.
- 로고, 문서명, 보안 등급, 페이지 번호 등 필요한 정보만 배치한다.
- 머리말과 바닥글에 과도한 장식 요소를 넣지 않는다.
- 표지에는 본문과 다른 머리말·바닥글을 적용할 수 있다.
- 구역이 나뉘는 경우 이전 구역과 연결 여부를 확인한다.
- 페이지 번호는 문서 전체에서 일관된 위치와 형식을 유지한다.
- 목차와 본문의 페이지 번호 체계가 다른 경우 구역을 분리하여 관리한다.

---

## 7. Image and Object Placement Rules

- 이미지는 페이지 폭을 넘어가지 않도록 크기를 조정한다.
- 사진, 스크린샷 및 공식 로고를 제외한 텍스트와 표는 가능한 한 편집 가능한 형태로 작성한다.
- 이미지와 도형은 본문 흐름을 깨지 않도록 배치한다.
- 일반 본문에서는 `텍스트와 같은 줄` 또는 안정적인 배치 방식을 우선 사용한다.
- 자유 배치 개체를 사용할 경우 편집 과정에서 위치가 밀리지 않는지 확인한다.
- 캡션, 출처 또는 설명이 필요한 이미지는 이미지와 함께 이동하도록 구성한다.
- 이미지 해상도를 과도하게 낮춰 인쇄 시 흐려지지 않도록 한다.

---

## 8. Editable Output Rules

- 텍스트는 편집 가능한 텍스트로 작성한다.
- 표는 편집 가능한 Word 표로 작성한다.
- 제목과 본문은 스타일을 사용하여 관리한다.
- 페이지 번호, 목차 및 캡션은 가능한 경우 자동 기능을 사용한다.
- 문서 전체를 통이미지로 만들지 않는다.
- 표지의 핵심 텍스트도 가능한 한 편집 가능한 상태로 유지한다.
- 문서 수정 후 목차, 페이지 번호, 상호 참조 및 개정 이력을 다시 확인한다.

---

## 9. Review Checklist

### Page Layout

- [ ] 기본 용지와 페이지 방향이 문서 목적에 맞는가?
- [ ] 본문, 표, 그림 및 주석이 인쇄 가능 영역 안에 배치되었는가?
- [ ] 불필요한 페이지 나누기나 구역 나누기가 남아 있지 않은가?
- [ ] 페이지 하단에 제목만 단독으로 남아 있지 않은가?

### Table Stability

- [ ] 표가 페이지 폭을 넘어가지 않는가?
- [ ] 표의 행이 페이지 사이에서 읽기 어렵게 분리되지 않는가?
- [ ] 여러 페이지 표의 머리글 행이 반복되는가?
- [ ] 표 안의 텍스트가 잘리거나 겹치지 않는가?
- [ ] 표를 한 페이지에 맞추기 위해 글자 크기를 과도하게 줄이지 않았는가?

### Blank Pages and Flow

- [ ] 불필요한 빈 페이지가 생성되지 않았는가?
- [ ] 숨겨진 빈 문단이나 중복된 나누기가 남아 있지 않은가?
- [ ] 표 뒤의 잔여 문단 때문에 빈 페이지가 생기지 않았는가?
- [ ] 페이지 방향 변경 구간 전후에 빈 구역이 남아 있지 않은가?

### Structure and Readability

- [ ] 제목, 본문, 표 및 주석의 계층이 명확한가?
- [ ] 제목 수준과 번호 체계가 일관적인가?
- [ ] 긴 문단이 적절히 분리되어 있는가?
- [ ] 문단 간격이 빈 문단이 아닌 서식으로 관리되는가?

### Brand and Editability

- [ ] 공통 브랜드 컬러, 로고, 타이포그래피 및 Do/Don't 기준을 따랐는가?
- [ ] 필요한 경우 Word 템플릿 자료를 함께 확인했는가?
- [ ] 텍스트와 표가 수정 가능한 형태로 구성되었는가?
- [ ] 문서 수정 후 목차, 페이지 번호 및 개정 이력을 다시 확인했는가?
